Vue d'ensemble du projet

L'outil analyse un fichier .apk Android en 7 étapes pour détecter les vulnérabilités de sécurité.

1/7
Analyse Statique
Décompile l'APK avec JADX, scanne le code Java avec des regex
Base
2/7
Analyse Dynamique
Envoie de vraies requêtes HTTP pour tester login/logout
Base
3/7
Cookie Scanner
Vérifie les flags OWASP sur les cookies HTTP (HttpOnly, Secure, SameSite)
Ajouté
4/7
Storage Scanner
Détecte les tokens stockés en clair (SharedPreferences, SQLite, logs)
Ajouté
5/7
Session Scanner
Teste entropie SID, fixation de session, timeout, invalidation au logout
Ajouté
6/7
Analyse IA (Claude)
Génère la checklist MASVS et résumé exécutif via l'API Anthropic
Base
7/7
Rapport PDF + JSON
Génère les rapports finaux dans le dossier results/

Installer JADX et l'ajouter au PATH
ℹ️
JADX est obligatoire pour l'étape 1. Sans lui, la décompilation APK échoue.
  • 1
    Télécharger jadx-1.5.5-no-jre-win.zip depuis github.com/skylot/jadx/releases
  • 2
    Extraire dans C:\Users\VotreNom\Downloads\jadx-1.5.5\
  • 3
    Ouvrir Variables d'environnement → PATH → Ajouter C:\Users\VotreNom\Downloads\jadx-1.5.5\bin
  • 4
    Ouvrir un nouveau terminal PowerShell et vérifier :
PowerShell
jadx --version
# Doit afficher : 1.5.5
⚠️
Si "jadx n'est pas reconnu" : Rechargez le PATH sans fermer PowerShell :
$env:PATH = [System.Environment]::GetEnvironmentVariable("PATH","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("PATH","User")

Erreurs courantes et solutions
● "jadx n'est pas reconnu"

Terminal pas rechargé après modification du PATH. Fermez et rouvrez PowerShell. Ou rechargez avec :

PowerShell
$env:PATH = [System.Environment]::GetEnvironmentVariable("PATH","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("PATH","User")
● "No module named 'fpdf'" ou 'anthropic'

Packages non installés :

PowerShell
pip install fpdf2 anthropic colorama requests python-dotenv
● "SyntaxError: Missing parentheses in call to 'print'"

Le fichier app.py original est en Python 2. Utiliser la version app.py corrigée fournie avec ce projet.

● "No module named 'flask_sqlalchemy'"

Incompatible avec Python 3.14. Utiliser la version simplifiée de app.py fournie — elle n'utilise pas SQLAlchemy, seulement Flask.

● "Backend inaccessible" à l'étape 2

Le serveur Flask n'est pas lancé ou a planté. Vérifier que Terminal 1 affiche "Running on http://127.0.0.1:8888" avant de lancer l'analyse.

● "Erreur IA : No module named 'anthropic'"

Installer le module et créer le fichier .env :

PowerShell
pip install anthropic
echo "ANTHROPIC_API_KEY=sk-ant-VOTRE-CLE" > .env
● "Aucun cookie intercepté" à l'étape 3

Normal sans mitmproxy configuré avec l'émulateur. Utiliser --demo pour la présentation — les résultats sont identiques visuellement.
